## 🎯 数据源配置

### 位置
`config.json` → `DATA_SOURCES` 数组

### ✅ 标准格式

每个数据源必须包含以下字段：

```json
{
    "url": "数据源的完整URL地址",
    "enabled": true,
    "name": "显示名称"
}
```

#### 字段说明

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| `url` | string | ✅ | - | 数据源URL（支持TXT/CSV格式） |
| `enabled` | boolean | ❌ | true | 是否启用该数据源 |
| `name` | string | ❌ | URL | 在日志中显示的名称 |

## ➕ 如何添加新数据源

### 步骤1: 打开配置文件

使用文本编辑器打开 `config.json`

### 步骤2: 找到 DATA_SOURCES 数组

搜索关键字：`"DATA_SOURCES": [`

### 步骤3: 添加新的数据源对象

在数组中添加新的数据源，**注意逗号分隔**：

#### 完整示例

```json
{
    "其他配置": "...",
    
    "DATA_SOURCES": [
        
        {
            "url": "https://example.com/source1.txt",
            "enabled": true,
            "name": "数据源1"
        },
 
        
        {
            "url": "https://example.com/source2.csv",
            "enabled": true,
            "name": "数据源2"
        },
 
        
        
        {
            "url": "https://raw.githubusercontent.com/用户名/仓库名/main/文件.txt",
            "enabled": true,
            "name": "GitHub数据源"
        }
    
    ]
}
```

### ⚠️ 重要注意事项

1. **逗号规则**: 
   - 数组元素之间**必须**用 `,` 分隔
   - **最后一个元素后面不要加逗号**

2. **启用/禁用**:
   ```json
   {"url": "...", "enabled": false, "name": "临时禁用"}
   ```

3. **URL要求**:
   - 必须是可直接访问的HTTP(S)链接
   - 返回内容为纯文本（TXT）或CSV格式
   - 不需要认证或特殊权限

## 📊 支持的数据格式

程序自动识别 **5种格式**：

### 格式1: 完整标准格式 ⭐推荐
```
104.16.132.229:443#US
```
✅ 包含：IP + 端口 + 国家代码

### 格式2: IP+端口
```
104.16.132.229:443
```
✅ 自动补充：默认国家代码 `#US`

### 格式3: 纯IP地址
```
104.16.132.229
```
✅ 自动补充：端口 `:443` + 国家 `#US`

### 格式4: CSV格式（逗号分隔）
```csv
IP,cf-meta-ip,端口,速度(Mbps),CF归属国,机房,...
40.233.68.138,40.233.68.138,443,165.68,CA,YYZ,30.16,40.88
```
✅ 自动处理：
- 跳过表头行（第一行包含"IP"）
- 提取：第1列(IP) + 第3列(端口) + 第5列(国家)

### 格式5: 注释和空行
```
# 这是注释行（自动跳过）

104.16.132.229:443#US

# 另一个注释
```
✅ 自动忽略以 `#` 开头的行和空行

## 🛡️ IP去重机制

### 多级去重策略

本工具采用 **三级去重** 确保最终结果无重复：

#### 第一级：获取时去重
- **位置**: `fetch_nodes()` 函数
- **原理**: 使用字典 `ip_node_map` 记录每个IP的最佳节点
- **效果**: 同一IP只保留第一个出现的节点

```python
ip_node_map = {}  # IP -> 节点映射
for node in nodes:
    ip = 提取IP(node)
    if ip not in ip_node_map:
        ip_node_map[ip] = node  # 只记录第一次出现
```

#### 第二级：集合去重
- **位置**: 获取完成后
- **原理**: 使用Python的 `set()` 数据结构
- **效果**: 自动去除完全重复的节点字符串

```python
all_nodes = set()  # 集合自动去重
all_nodes.update(nodes)
```

#### 第三级：输出前去重
- **位置**: 最终保存前
- **原理**: 再次遍历检查IP唯一性
- **效果**: 双重保险，确保100%无重复

```python
final_unique = []
seen_ips = set()
for node in final_selected:
    ip = node.split(':')[0]
    if ip not in seen_ips:
        seen_ips.add(ip)
        final_unique.append(node)
```

### 去重效果示例

**输入** (假设从多个数据源获取):
```
数据源A: 104.16.132.229:443#US
数据源B: 104.16.132.229:443#US  ← 重复！
数据源C: 104.16.133.230:443#US
```

**输出** (去重后):
```
104.16.132.229:443#US  ✓ 保留
104.16.133.230:443#US  ✓ 保留
总计: 2个唯一IP (去除1个重复)
```

### 日志输出

运行时会显示去重统计：
```
==================================================
原始节点: 5000 个
IP去重后: 3200 个 (去除 1800 个重复IP)

总计获取 3200 个唯一IP节点
==================================================
```

## ⚡ 性能优化建议

## ❓ 常见问题

### Q1: 添加数据源后没有生效？
**检查清单**:
- [ ] JSON格式是否正确（可用在线验证器）
- [ ] 逗号和括号是否匹配
- [ ] `enabled` 字段是否为 `true`
- [ ] URL是否可直接在浏览器中访问

### Q2: 如何验证数据源是否工作？
运行程序查看日志：
```bash
python main.py
```

**预期输出**:
```
开始从 16 个数据源获取节点...
============================================================
  正在获取 [你的新数据源] ... [OK] XXX 个节点  ← 成功
============================================================
```

### Q3: CSV文件格式有特殊要求吗？
- 必须包含表头行（第一行为列名）
- IP必须在第1列
- 端口在第3列（可选）
- 国家代码在第5列（可选）

### Q4: 如何临时禁用某个数据源？
将 `"enabled": true` 改为 `"enabled": false`

### Q5: 去重后会丢失重要信息吗？
不会。去重仅针对**相同IP**的节点：
- 如果同一IP出现在多个数据源，只保留**第一个**
- 不同IP的节点全部保留
- 最终结果保证：快速、稳定、低延迟、**不重复**

## 🔍 故障排查

### 问题: 数据源获取失败

**可能原因及解决方案**:

1. **URL不可访问**
   - 浏览器测试URL是否正常
   - 检查网络连接

2. **格式不兼容**
   - 确认返回的是纯文本而非HTML
   - 检查是否需要登录认证

3. **被防火墙拦截**
   - 尝试更换网络环境
   - 检查代理设置

4. **超时**
   - 增加 `FETCH_TIMEOUT` 配置值
   - 将大文件数据源拆分
